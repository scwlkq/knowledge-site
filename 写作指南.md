# 写作指南

本文档介绍如何在这个知识站中创建和发布新文章。

## 快速开始

### 方式一：使用 Hugo 命令创建（推荐）

```bash
# 进入项目目录
cd /Users/suncong32/Desktop/github/knowledge-site

# 创建新文章
hugo new content/posts/文章标题.md

# 编辑文章
# 使用你喜欢的编辑器打开 content/posts/文章标题.md
```

Hugo 会自动生成带有 Front Matter 的文章模板。

### 方式二：手动创建文件

直接在 `content/posts/` 目录下创建 `.md` 文件，例如 `my-article.md`。

## 文章结构

每篇文章由两部分组成：

1. **Front Matter**（文章元数据）- 位于文件开头，用 `---` 包裹
2. **正文内容** - Markdown 格式

### 完整示例

```markdown
---
title: "我的文章标题"
date: 2026-05-24T16:50:00+08:00
draft: false
tags: ["技术", "学习", "笔记"]
categories: ["编程"]
description: "这是文章的简短描述"
```

## Front Matter 字段说明

| 字段 | 必填 | 说明 | 示例 |
|------|------|------|------|
| `title` | ✅ | 文章标题 | `"我的第一篇文章"` |
| `date` | ✅ | 发布日期时间 | `2026-05-24T16:50:00+08:00` |
| `draft` | ✅ | 是否为草稿 | `false`（发布）/ `true`（草稿） |
| `tags` | ❌ | 标签列表 | `["技术", "学习"]` |
| `categories` | ❌ | 分类列表 | `["编程", "笔记"]` |
| `description` | ❌ | 文章描述（SEO） | `"这是一篇关于..."` |
| `author` | ❌ | 作者（覆盖默认） | `"sc"` |
| `weight` | ❌ | 排序权重（数字越小越靠前） | `1` |

### 重要提示

- **`draft: false`** - 文章才会发布到线上
- **`draft: true`** - 文章只在本地预览时显示（使用 `hugo server -D`）

## 本地预览

在发布前，先在本地预览效果：

```bash
# 进入项目目录
cd /Users/suncong32/Desktop/github/knowledge-site

# 启动本地服务器（包含草稿）
hugo server -D

# 或者只预览非草稿文章
hugo server
```

访问 http://localhost:1313 查看效果。

按 `Ctrl+C` 停止服务器。

## 发布流程

### 1. 确认文章状态

确保 `draft: false`，否则不会发布到线上。

### 2. 提交到 Git

```bash
# 查看修改
git status

# 添加文件
git add .

# 提交
git commit -m "添加新文章：文章标题"

# 推送到 GitHub
git push
```

### 3. 自动部署

推送后，GitHub Actions 会自动：
1. 检测到代码变更
2. 构建 Hugo 站点
3. 部署到 GitHub Pages

整个过程需要 2-5 分钟。

### 4. 查看结果

访问 https://scwlkq.github.io/knowledge-site/ 查看发布的文章。

## 文件命名建议

### 推荐的命名方式

```
content/posts/
├── 2026-05-24-first-post.md          # 带日期前缀
├── hugo-tutorial.md                   # 英文短横线分隔
├── 学习笔记-python基础.md             # 中文也可以
└── tech/                              # 可以创建子目录分类
    └── docker-guide.md
```

### 命名规范

- ✅ 使用小写字母
- ✅ 用短横线 `-` 分隔单词
- ✅ 避免空格和特殊字符
- ✅ 文件名要有意义（便于管理）
- ✅ 可以使用中文（但英文更通用）

## 常见问题

### Q: 文章写完了但网站上看不到？

A: 检查以下几点：
1. `draft: false` 了吗？
2. `git push` 了吗？
3. GitHub Actions 构建完成了吗？（查看仓库的 Actions 标签页）
4. 等待 2-5 分钟让部署完成

### Q: 本地预览正常，但发布后样式不对？

A: 检查 `hugo.toml` 中的 `baseURL` 是否正确设置为：
```toml
baseURL = 'https://scwlkq.github.io/knowledge-site/'
```

### Q: 如何修改已发布的文章？

A: 直接编辑对应的 `.md` 文件，然后 `git commit` 和 `git push` 即可。

### Q: 如何删除文章？

A: 删除对应的 `.md` 文件，然后提交：
```bash
git rm content/posts/要删除的文章.md
git commit -m "删除文章：XXX"
git push
```

### Q: 可以插入图片吗？

A: 可以。有两种方式：
1. 使用外部图床（如 GitHub、图床服务）
2. 将图片放在 `static/images/` 目录，引用时用 `/images/图片名.jpg`

示例：
```markdown
![示例图片](/images/example.jpg)
```

### Q: 如何设置文章置顶？

A: 在 Front Matter 中添加 `weight` 字段，数字越小越靠前：
```yaml
---
title: "置顶文章"
weight: 1
---
```

## 进阶技巧

### 使用草稿功能

写作过程中，设置 `draft: true`，可以在本地预览但不会发布：

```bash
hugo server -D  # -D 参数显示草稿
```

完成后改为 `draft: false` 再发布。

### 批量创建文章

```bash
# 创建多篇文章
hugo new content/posts/article1.md
hugo new content/posts/article2.md
hugo new content/posts/article3.md
```

### 使用模板

编辑 `archetypes/default.md` 自定义文章模板：

```markdown
---
title: "{{ replace .File.ContentBaseName "-" " " | title }}"
date: {{ .Date }}
draft: true
tags: []
categories: []
---

文章开始...
```

之后用 `hugo new` 创建的文章都会使用这个模板。

## 写作建议

1. **标题清晰** - 让读者一眼知道文章内容
2. **结构分明** - 使用标题层级组织内容
3. **代码高亮** - 指定代码语言以获得语法高亮
4. **适当配图** - 图文并茂更易理解
5. **添加标签** - 方便分类和检索
6. **定期发布** - 保持更新频率

## 快捷命令

将以下命令添加到 shell 配置文件（`~/.zshrc` 或 `~/.bashrc`）：

```bash
# Hugo 知识站快捷命令
alias blog='cd /Users/suncong32/Desktop/github/knowledge-site'
alias blog-new='cd /Users/suncong32/Desktop/github/knowledge-site && hugo new content/posts/'
alias blog-serve='cd /Users/suncong32/Desktop/github/knowledge-site && hugo server -D'
alias blog-publish='cd /Users/suncong32/Desktop/github/knowledge-site && git add . && git commit -m "更新文章" && git push'
```

使用：
```bash
blog          # 进入项目目录
blog-new 文章名.md   # 创建新文章
blog-serve    # 启动本地预览
blog-publish  # 一键发布
```
